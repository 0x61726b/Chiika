# (void) preloadUIItems()

The purpose of this method is to load UI views and their respective databases Why 'preload' is, I wanted to delay script compilation before UI is loaded So scripts can access UI elements without subscribing to a event. I think it worts the trade off between loading time, which doesn't affect much @return

# (void) checkUIData()

Will check if there are empty 'dataSource's. If one is found, we will mark it needUpdate then update method will call the script and there the script can fill data If the data source meant to be empty, just ignore the callback on the script end @return
